# 一、什么是异步线程池

## 1. 核心概念

**线程池**：提前创建一批工作线程，复用线程、避免频繁创建销毁线程带来的开销。

**异步**：把任务交给线程池去执行，主线程不用阻塞等待结果，可以继续往下执行后续代码，等任务执行完成后再获取结果。

Java 中核心类：

- `Executor`：顶层接口
- `ExecutorService`：继承 Executor，提供提交任务、关闭线程池方法
- `ThreadPoolExecutor`：手动自定义线程池（推荐）
- `Executors`：工具类，快速生成线程池（不推荐生产直接用）
- `CompletableFuture`：实现**异步编程**，搭配线程池做异步任务。

# 二、ThreadPoolExecutor 七大核心参数（重中之重）

```java
public ThreadPoolExecutor(
    int corePoolSize,      // 1.核心线程数（常驻线程，空闲也不回收）
    int maximumPoolSize,   // 2.最大线程数
    long keepAliveTime,    // 3.非核心线程空闲存活时间
    TimeUnit unit,         // 4.时间单位
    BlockingQueue<Runnable> workQueue, // 5.阻塞队列，存放等待的任务
    ThreadFactory threadFactory,       // 6.线程工厂，用来创建线程、设置线程名
    RejectedExecutionHandler handler   // 7.拒绝策略
)
```

## 执行流程（任务提交顺序）

1. 来了任务，判断当前运行线程数是否＜**核心线程数**：小于则新建核心线程执行任务；
2. 已经达到核心线程数 → 任务放入**阻塞队列**排队；
3. 队列满了 → 判断当前线程数是否小于最大线程数，新建非核心线程处理任务；
4. 线程数达到最大线程数、队列也满了 → 执行**拒绝策略**。

> 顺序总结：核心线程 → 阻塞队列 → 非核心线程 → 拒绝策略

## 1）四个拒绝策略

1. `AbortPolicy`（默认）：直接抛异常`RejectedExecutionException`
2. `CallerRunsPolicy`：让调用任务的主线程自己去执行任务
3. `DiscardPolicy`：直接丢弃任务，不报错
4. `DiscardOldestPolicy`：丢弃队列最老的任务，放入新任务

# 三、Executors 提供的 4 种默认线程池（生产不推荐，容易 OOM）

（Out Of Memory，内存溢出程序申请内存时，虚拟机没有足够可用内存分配，直接抛出内存溢出错误，程序崩溃）

1. **newFixedThreadPool**（固定线程池）
    
    - 核心线程 = 最大线程，无空闲回收；队列是无界队列`LinkedBlockingQueue`
    - 问题：任务无限堆积，内存溢出 OOM
    
2. **newCachedThreadPool**（缓存线程池）
    
    - 核心线程 0，最大线程无限；非核心线程 60s 回收
    - 问题：任务突增会创建无限多线程，CPU 耗尽
    
3. **newSingleThreadExecutor**（单线程池）
    
    - 只有 1 个核心线程，无界队列，任务无限堆积 OOM
    
4. **newScheduledThreadPool**（定时线程池）
    
    - 用于定时、延迟任务，最大线程数无限制，同样有 OOM 风险
    

> 生产规范：禁止直接使用 Executors，必须手动 new ThreadPoolExecutor 自定义线程池，限制队列长度、线程数量。

# 四、异步实现：CompletableFuture + 自定义线程池

CompletableFuture 用来做异步回调，脱离主线程执行，实现非阻塞。

```java
//自定义线程池
ThreadPoolExecutor pool = new ThreadPoolExecutor(
        5,
        10,
        1L,
        TimeUnit.SECONDS,
        new ArrayBlockingQueue<>(20),
        Executors.defaultThreadFactory(),
        new ThreadPoolExecutor.CallerRunsPolicy()
);

//执行异步任务
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    //异步执行业务逻辑
    return "任务结果";
}, pool);

future.thenAccept(res->{
    //异步回调，任务执行完之后执行
    System.out.println(res);
});
```

- `supplyAsync`：有返回值异步任务
- `runAsync`：无返回值异步任务
- `thenApply`：拿到上一步结果，再处理并返回新值
- `thenCombine`：两个异步任务并行执行，等两个都完成后汇总结果

# 五、核心调参经验（生产怎么设置核心线程数）

1. **IO 密集型**（数据库、Redis、HTTP 请求，线程大部分时间在等待）
    
    公式：`核心线程数 = 2 * CPU核心数`，甚至可以更大，因为线程在等待不占用 CPU
2. **CPU 密集型**（大量计算、循环逻辑）
    
    公式：`核心线程数 = CPU核心数 + 1`，避免线程上下文频繁切换损耗性能

# 六、线程池的四种状态

1. RUNNING：运行状态，接收新任务，处理队列任务
2. SHUTDOWN：调用 shutdown ()，不再接收新任务，处理完已有队列任务
3. STOP：调用 shutdownNow ()，不接收新任务，中断正在执行的任务，丢弃队列任务
4. TERMINATED：所有线程销毁，线程池彻底关闭

# 七、使用注意事项

1. 必须手动定义线程池，指定有界队列，防止 OOM；
2. 给线程自定义线程名，便于日志排查问题；
3. 项目关闭时调用 shutdown 优雅关闭线程池，防止内存泄漏；
4. 异步任务内部一定要捕获异常，否则异常会丢失，排查困难。
